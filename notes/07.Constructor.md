# Constructors

**Definition:**  
A constructor is a special method used to **initialize objects**.

- Same name as **class name**
- No return type (not even void)
- Called automatically when object is created

---

## Object Creation

Student s = new Student();

- **Student s** → reference variable  
- **new Student()** → creates object  
- Constructor is invoked here  

---

## Syntax

ClassName(type param1, type param2, ...) {
    // initialization code
}

---

## Example

class Student {
    int id;

    Student(int newId) {
        id = newId;
    }
}

---

## Object Creation Example

Student s = new Student(1001);

---

## Default Constructor

- If constructor is NOT provided  
- Compiler automatically creates one  

Student() {
    // no arguments constructor
}

---

## Important Rule

- If you create ANY constructor  
  → Default constructor is NOT created  

---

## Example (Important)

class Student {
    int id;

    Student(int newId) {
        id = newId;
    }
}

Student s1 = new Student(1001);  // ✅ valid
Student s2 = new Student();      // ❌ error

---

## Key Points

- Constructor name = class name  
- Used to initialize object  
- No return type  
- Automatically called  

---

## Memory Concept

Student s = new Student();

- Memory allocated for object  
- Reference variable stores address  
- Constructor initializes object  


# Constructor Overloading

**Definition:**  
Using multiple constructors in the same class with **different parameter lists**.

- Same constructor name (class name)
- Different parameters
- Improves **flexibility and convenience**

---

## Why Constructor Overloading?

- Provides **convenience**
- Allows creating objects in different ways
- Avoids multiple class names

---

## Rules

- Constructor name must be same as class name  
- Parameters must be different:
  - Different number of parameters  
  - Different data types  
  - Different order  

---

## Example

FileOutputStream(String name, boolean append)

FileOutputStream(String name)

FileOutputStream(File file)

FileOutputStream(File file, boolean append)

FileOutputStream(FileDescriptor fdObj)

---

## Custom Example

class Student {
    int id;

    Student() {
        id = 0;
    }

    Student(int newId) {
        id = newId;
    }
}

Student s1 = new Student();      // uses no-arg constructor
Student s2 = new Student(1001);  // uses parameterized constructor

---

## Important Points

- Constructor overloading = **compile-time polymorphism**
- Helps create objects with different initial values
- Same class → multiple ways to initialize  


# this Reference

**Definition:**  
`this` refers to the **current object** of the class.

- Used to access **instance variables**
- Used to call **constructors**
- Helps avoid confusion between variables

---

## Why use `this`?

When parameter names and instance variables are same:

class Student {
    int id;

    Student(int id) {
        this.id = id;   // refers to instance variable
    }
}

---

## Constructor Chaining

- `this()` is used to call another constructor in the same class
- Must be the **first statement**

Example:

Student(int id, String name, String gender, int age, long phone, double gpa, char degree) {
    this(id, name, gender, age, phone, gpa, degree, false);
}

Student(int id, String name, String gender, int age, long phone, double gpa, char degree, boolean international) {
    this.id = id;
    this.name = name;
    this.gender = gender;
    this.age = age;
    this.phone = phone;
    this.gpa = gpa;
    this.degree = degree;
    this.international = international;
}

---

## Real Example (from your code)

this.id = id;
this.name = name;
this.gender = gender;
this.age = age;
this.phone = phone;
this.gpa = gpa;
this.degree = degree;

---

## Important Points

- `this` refers to **current object**
- Used to differentiate:
  variable vs parameter  
- `this()` → calls constructor  
- Cannot be used in static context  

---

## Easy Trick

- `this.variable` → instance variable  
- `variable` → local variable
